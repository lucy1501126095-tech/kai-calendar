# kai_calendar

Kai的日历系统 — AstrBot插件

## 功能

### 经期追踪
- 以黄体酮周期为核心逻辑（非自然周期预测）
- 记录黄体酮开始/停药、来经/结束日期
- 自动计算"停药→来经"响应天数
- 区分自然来经和药物催经
- 趋势分析：响应天数变化、自然周期出现、用药天数趋势

### 用药打卡
- 支持肌醇、镁片、K2、黄体酮四项
- Streak连续打卡计数（断了归零，保留历史最佳含起止日期）
- 缺失天数追踪（配合主动提醒升级：1天温柔→3天严肃→7天报警）

### 身体备注
- 随时记录身体状况（腹痛、情绪波动、出血量等）
- 按时间范围查询

### 纪念日
- 支持阳历和农历（依赖zhdate库）
- 每年自动转换农历日期
- 预存纪念日（见面1214/求婚1219/结婚1225/每月14号/宝宝农历生日）

### Kai备注栏
- Kai的私人笔记，主动写入
- 宝宝不问不说

### 待办提醒
- 添加/完成/删除/列表
- 支持截止日期

### 中国法定假日
- 内置固定假日 + 农历假日自动转换
- 清明节日期查表

### 数据备份
- 手动备份（/cal_backup指令或工具调用）
- 插件关闭时自动备份
- 保留最近4份，自动清理旧备份

## 技术架构

- **存储**: SQLite内嵌，Python自带sqlite3，零外部依赖
- **工具注册**: @llm_tool装饰器，主模型在对话中自主调用
- **system prompt**: 只有固定的工具描述文字，不注入动态数据，不影响缓存命中
- **主动提醒**: 通过cal_daily_check工具配合kai_proactive定时任务

## 安装

### 依赖
- 农历支持需要安装zhdate: `pip install zhdate`
- 不安装zhdate时农历功能自动禁用，其他功能正常

### 安装方式
在AstrBot管理面板添加插件仓库地址：
```
https://github.com/lucy1501126095-tech/kai-calendar
```

## 指令

- `/cal_backup` — 手动触发数据备份
- `/cal_status` — 查看系统状态

## 工具列表（function calling）

| 工具名 | 功能 |
|--------|------|
| cal_period_record | 记录经期（黄体酮开始/停药/来经/结束） |
| cal_period_query | 查询经期记录和趋势 |
| cal_medication | 记录用药打卡 |
| cal_medication_query | 查询用药记录和streak |
| cal_body_note | 记录/查询身体备注 |
| cal_anniversary | 管理纪念日（增删改查） |
| cal_kai_note | Kai私人备注（写入/读取） |
| cal_todo | 管理待办（增删改查） |
| cal_holidays | 查询中国法定假日 |
| cal_daily_check | 每日综合检查 |
| cal_backup | 备份管理 |
